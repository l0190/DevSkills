# Judge — 判断

**目的**：基于观察数据做出判断。

## 判断内容

1. **问题是否存在**：观察数据是否显示异常？
2. **问题是否已定位**：是否找到了根本原因？
3. **假设是否成立**：上一轮的假设是否被观察数据支持或否定？

## 核心要求

**判断必须引用具体的观察数据**，不能用"感觉"或"可能"。推理过程要清晰展示从数据到结论的因果链。

## 输出格式

```markdown
### 判断 #N

**基于观察**：[引用具体的观察记录编号和数据]
**判断结果**：
- [ ] 问题存在 / 问题不存在
- [ ] 根因已定位 / 需要继续排查
- [ ] 假设成立 / 假设不成立 / 需要新假设

**推理过程**：
[数据A的值是X，预期应该是Y → 说明Z环节出了问题 → ...]
```

## 常见判断模式

| 观察到的现象 | 可能的结论 |
|-------------|-----------|
| 变量值为 None | 上游函数返回了 None，或初始化缺失 |
| 索引越界 | 循环范围或列表长度计算有误 |
| 类型不匹配 | 数据转换或类型标注有误 |
| 值与预期偏差固定比例 | 常量或系数错误 |
| 值与预期偏差不规律 | 条件分支逻辑错误 |
| git diff 显示某行被改过 | 该改动可能是引入点 |
