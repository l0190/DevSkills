# Decide — 决策

**目的**：基于判断，明确下一步行动。

## 核心要求

**铁律：没有观察依据，不做决策。**

决策必须包含三个要素：
1. **要做什么** — 具体的操作描述
2. **为什么做** — 基于哪个观察/判断得出的结论
3. **预期结果** — 执行后预期会观察到什么

## 输出格式

```markdown
### 决策 #N

**观察依据**：[引用观察记录编号]
**操作**：[具体的下一步操作]
**原因**：[为什么执行这个操作]
**预期观察**：[执行后预期看到什么结果]
```

## 决策类型

| 类型 | 场景 | 示例 |
|------|------|------|
| 继续观察 | 当前信息不足以定位问题 | "在函数入口处添加插桩，追踪参数值" |
| 提出假设并验证 | 有初步判断需要验证 | "假设是索引越界，添加边界检查日志验证" |
| 执行修复 | 根因已定位 | "将 `range(len+1-n)` 改为 `range(n)`" |
| 缩小范围 | 问题区域太大 | "用 git diff 确认哪个 commit 引入了 bug" |
| 切换方向 | 当前假设被否定 | "排除类型错误假设，转而检查数据流" |

## 注意事项

- **每次只验证一个假设**：同时改变多处，无法确定哪处是关键
- **预期要具体可验证**："代码不报错"太模糊，"函数返回 `[95, 92, 91]`" 才是好的预期
