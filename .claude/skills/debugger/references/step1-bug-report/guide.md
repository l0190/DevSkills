# Step 1: 生成 Bug 报告

调试开始前，先将问题结构化为 Bug 报告。报告是整个调试过程的基础。

## 输入来源（二选一或兼有）

1. **用户提供**：错误堆栈、异常信息、测试失败输出、异常行为描述
2. **自动检测**：运行程序/测试，捕获异常/断言失败/输出异常

## 收集流程

1. 从用户输入中提取：错误类型、错误消息、完整堆栈跟踪
2. 定位错误发生的文件和行号
3. 阅读错误点附近的源码，识别数据流路径
4. 明确调试目标：什么算"修好了"、如何验证

## 报告模板

```markdown
# Bug Report: [简短问题摘要]

## 1. 问题描述
- **现象**：[用户报告的错误现象]
- **期望行为**：[代码应该做什么]
- **实际行为**：[代码实际做了什么]

## 2. 错误信息
- **错误类型**：[ErrorType]
- **错误消息**：[完整错误消息]
- **堆栈跟踪**：
\`\`\`
[完整堆栈]
\`\`\`

## 3. 环境信息
- **触发方式**：[运行命令/测试命令]
- **Python版本**：[如已知]
- **相关依赖**：[如已知]

## 4. 初步分析
- **错误定位**：`[file]:[line]`
- **可疑点**：[基于堆栈和代码的初步判断]
- **数据流路径**：[从触发点到错误点的调用链]

## 5. 调试目标
- **目标**：[明确什么算"修好了"]
- **验证方式**：[如何验证修复有效]
```
