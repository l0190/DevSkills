# Objective — 确认目标

**目的**：基于观察数据，确认本轮调试的具体目标。

## 确认目标内容

1. **问题是否存在**：观察数据是否显示异常？
2. **问题是否已定位**：是否找到了根本原因？
3. **假设是否成立**：上一轮的假设是否被观察数据支持或否定？
4. **本轮目标**：明确本轮 OODA 循环需要达成什么

## 核心要求

**目标确认必须引用具体的观察数据**，不能用"感觉"或"可能"。推理过程要清晰展示从数据到结论的因果链。

## 输出格式

```markdown
### 目标确认 #N

**基于观察**：[引用具体的观察记录编号和数据]
**目标确认**：
- [ ] 问题存在 / 问题不存在
- [ ] 根因已定位 / 需要继续排查
- [ ] 假设成立 / 假设不成立 / 需要新假设

**本轮目标**：[明确本轮需要验证的假设或达成的结果]

**推理过程**：
[数据A的值是X，预期应该是Y → 说明Z环节出了问题 → ...]
```

## 常见目标确认模式

| 观察到的现象 | 确认的目标 |
|-------------|-----------|
| 变量值为 None | 确认上游函数为何返回 None |
| 索引越界 | 确认循环范围或列表长度计算是否有误 |
| 类型不匹配 | 确认数据转换或类型标注是否有误 |
| 值与预期偏差固定比例 | 确认常量或系数是否正确 |
| 值与预期偏差不规律 | 确认条件分支逻辑是否正确 |
| git diff 显示某行被改过 | 确认该改动是否为引入点 |
