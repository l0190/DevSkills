# Act — 执行

**目的**：执行决策中的操作，收集新的数据。

## 执行操作类型

- 添加/修改插桩代码
- 修改源码（修复）
- 运行程序/测试
- 读取文件/分析数据
- 查阅文档/源码

## 输出格式

```markdown
### 执行 #N

**操作类型**：[插桩/修复/运行/分析]
**执行内容**：
[实际执行的操作描述和代码变更]

**执行结果**：
[运行输出、测试结果、文件内容等]
```

## 执行原则

- **保持代码可运行**：插桩后代码必须仍可执行
- **最小化插桩范围**：只在能提供关键信息的点插桩，避免日志洪泛
- **备份原文件**：修改前先备份，便于回退

## 代码规范

```python
# OBSERVE: [观察目的] - OODA迭代#[N]
print(f"[OBSERVE] func_name | var = {var}")

# FIX: [修复原因] - 基于OODA观察#[N]
[修复代码]
```

**前缀约定：**
- `# OBSERVE:` — 观察插桩（调试完成后应清理）
- `# FIX:` — 修复代码（永久保留）
- `[OBSERVE]` — 运行时日志前缀（便于 grep 过滤）

## 修复后验证

**必须实际运行代码验证**，不可仅靠代码走读：

- 如果用户提供了期望值，修复后的输出**必须精确匹配**
- 不匹配则继续 OODA 循环
- 修复完成后移除所有 `[OBSERVE]` 插桩代码

## 收敛检查

每轮 OODA 结束后执行：

```markdown
### 收敛检查

- 本轮是否有新发现：[是/否]
- 连续无新发现轮数：[N/3]
- 当前问题状态：[已定位/部分定位/未定位]
- 代码运行状态：[符合目标/仍有问题]
- 下一步：[继续OODA循环 / 输出调试报告]
```

**终止条件（满足任一即可）：**
1. 代码运行符合用户目标（期望值精确匹配）
2. 连续 3 轮无新发现
