# Step 3: 生成调试报告

调试结束后，生成完整的 Markdown 报告并保存。

## 报告模板

```markdown
# OODA 调试报告: [问题摘要]

## 摘要
- **问题**：[一句话描述]
- **根因**：[一句话描述根因]
- **修复方案**：[一句话描述修复]
- **OODA迭代轮数**：[N]
- **最终状态**：[已修复/部分修复/需要人工介入]

## Bug 报告
[嵌入 Step 1 生成的 Bug 报告]

## OODA 决策链

### 迭代 1
[嵌入 Observe #1 → Objective #1 → Decide #1 → Act #1 → 收敛检查]

### 迭代 2
[嵌入完整的 OODA 记录]

## 修复记录

### 最终修复
\`\`\`diff
[修复代码的 diff]
\`\`\`

### 代码修改清单
| 文件 | 修改类型 | 描述 |
|------|---------|------|
| file.py | 修复 | [描述] |

## 验证结果
- **验证命令**：[执行的验证命令]
- **输出**：[关键输出]
- **状态**：[通过/失败]

## 调试过程统计
- 总迭代轮数：[N]
- 使用过的观察手段：[列表]
- 提出的假设数：[N]
- 被否定的假设数：[N]
```

## 保存位置

```bash
REPORT_DATE=$(date +%Y-%m-%d)
REPORT_DIR="docs/debug/${REPORT_DATE}"
mkdir -p "${REPORT_DIR}"
```

- 文件名格式：`YYYY-MM-DD-NNN_<问题摘要>.md`
- 更新索引文件 `docs/debug/debug_index.md`，追加本次调试条目
