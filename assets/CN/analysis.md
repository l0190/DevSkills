AI进行软件开发分为两步 = 像做数学题一样构建伪代码(结构化描述文档) -> 将伪代码翻译成目标语言
- 结构化描述文档不应该包含具体语言,专注于描述项目本身
架构设计(接口+通信) -> 算法设计(伪代码) -> 编程语言
- 软件架构设计关注模块接口设计和模块间通信设计
- 这里的算法设计就是设计算法的大步骤(需要用注释标注其作用)
弄清楚自己的任务是算法开发(伪代码)还是应用工程(架构设计),两者解决任务的思路不同
- 应用工程:
	- 将源码解析任务转换成从源码中收集信息构建项目分析文档
- 算法开发:
	- 操作组织log,根据日志形成算法分析报告(函数调用链,关键变量的变化,输入,输出)
	- 算法追踪(trace):算法外部超参数、算法临时变量、传入处理变量


- 软件系统分析的第一步就是判断整个工程的入口
- 核心模块在软件系统时就要初始化.因此根据入口展开调用链,查找初始化就可以找到软件架构中的模块
    - 其他模块可以在启动后懒加载
- Python脚本生成函数调用图,要求如下:
	1. sys.setprofile进行函数trace
	2. 保留项目内的调用
	3. 从main开始进行trace
关键变量信息披露:小变量返回值,大变量输出shape()
